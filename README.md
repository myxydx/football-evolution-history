# 足球进化史

一个把足球历史做成“科技树”的交互式静态网页。

项目从 1848 年剑桥规则、1863 年英足总统一规则出发，把现代足球的发展拆成四条线：规则科技、战术角色、赛事商业、文化社会。用户可以像玩策略游戏一样点击节点、解锁后续内容，在一张时间树里理解足球为什么会从校园和地方俱乐部游戏，长成今天的全球运动、商业产业和文化现场。

在线版本：

<https://myxydx.github.io/football-evolution-history/>

## 玩法

- 在“研发模式”中，从早期节点开始点击“可研”卡片，逐步解锁后续足球史节点。
- 点击任意已出现的卡片，可以在右侧 wiki 查看背景介绍、真实案例、关联解释、图片和来源。
- 切换到“长时间轴”，可以按年份浏览完整节点。
- 使用缩放和底部进度条，可以在较大的科技树中快速定位。
- 节点之间的线条是策展关系：实线偏向推动关系，虚线多为时间参考或同类演进，不代表唯一历史因果。

## 四条分支

- 规则科技：越位、点球、红黄牌、门线技术、VAR、半自动越位、补水暂停等。
- 战术角色：2-3-5、WM、链式防守、全攻全守、高位逼抢、Tiki-taka、门卫型门将、位置足球等。
- 赛事商业：足总杯、职业联赛、世界杯、欧冠、英超商业化、博斯曼判决、金元足球、财政公平政策、沙特联赛金元等。
- 文化社会：英足总女足禁令、慕尼黑空难、足球流氓治理、马拉多纳与英阿记忆、Ultras 文化、梅罗时代球迷等。

## 项目结构

```text
.
├── index.html      # 页面结构和入口
├── styles.css      # UI、卡片、时间轴和响应式样式
├── data.js         # 节点、关联、wiki 文案、来源和图片信息
├── app.js          # 研发解锁、渲染、拖动、缩放、搜索等交互逻辑
├── assets/         # 本地图片资产
└── .nojekyll       # GitHub Pages 静态发布配置
```

项目不依赖构建工具，仍然是纯静态网页。拆分文件只是为了让内容和交互更容易维护。

## 内容口径

这是一个面向大众理解的足球史互动项目，不是严肃学术数据库。节点关系采用“策展式历史解释”：

- 硬事实节点尽量保留具体年份、机构、赛事和真实案例。
- 战术、文化和商业趋势按代表时期处理，部分年份是代表性时间点。
- “可研发”关系用于交互体验，不等于严格因果链。
- wiki 中的来源主要指向 FIFA、UEFA、IFAB、俱乐部官网、公开资料入口和 Wikimedia Commons 等。

如果继续正式发布，建议后续逐条复核图片授权、完善出处格式，并为重点节点增加更严格的参考资料。
